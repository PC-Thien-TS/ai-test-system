# EcommerceAlphaAdapter

This adapter was generated by `create_project_adapter.py` using profile `ecommerce`.

## What This Adapter Owns
- Product flow definitions
- Suite registry and priorities
- Defect family registry
- Change-aware flow mapping
- Risk/scoring defaults
- Blocker classification rules

## Generated Flows
- `auth_foundation`: Auth Foundation
- `catalog_discovery`: Catalog Discovery
- `cart_checkout`: Cart & Checkout
- `order_management`: Order Management
- `payment_integrity`: Payment Integrity
- `admin_ops`: Admin Operations

## Customization Checklist
1. Update flow IDs/titles/descriptions in `flow_registry.py` to match your real product.
2. Replace placeholder suites in `suite_registry.py` with runnable pytest suites.
3. Define real defect families in `defect_registry.py`.
4. Extend file-to-flow mapping in `change_mapping.py`.
5. Calibrate risk and scoring rules in `risk_rules.py`.
6. Refine blocker classification logic in `blocker_rules.py`.
7. Implement project-specific defect family construction in `adapter.py -> build_defect_families(...)`.

## Activation
Use this adapter by setting:

`AI_TESTING_ADAPTER=ecommerce_alpha`

Then run the existing platform commands, for example:

`python ai_regression_orchestrator.py --intent full_app_fast_regression --mode fast`

## Notes
- This scaffold is structurally valid and contract-compliant.
- It is not production-ready until you replace placeholder suites/rules with project evidence.
