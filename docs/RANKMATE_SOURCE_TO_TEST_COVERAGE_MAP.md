## High-Risk Cluster Summary (Wave 1)
- Payment + Order Creation + Order Lifecycle + Merchant Actions + Admin Monitoring.
- Token/session/idempotency boundaries in both user and merchant clients.
- Callback/webhook and transition idempotency paths.

## Gaps / Blockers Captured in Map
- Webhook signature fixture generation is required for reliable callback tests.
- Multi-role seeded data orchestration is required for stable cross-repo e2e.
- Notification real-time (SignalR) should be treated as separate wave unless needed for release gate.
